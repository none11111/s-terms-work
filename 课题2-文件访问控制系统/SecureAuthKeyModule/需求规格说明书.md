## 一、引言

### 1.1 编写目的

本文档旨在详细描述SecureAuthKeyModule安全认证与密钥管理模块的功能需求、非功能需求、数据需求和接口需求，为系统设计、开发和测试提供明确的依据，确保模块满足信息系统安全课程设计的要求。

### 1.2 项目背景

在文件访问控制系统中，用户认证和密钥管理是保障系统安全的核心组件。传统的密钥管理方式存在密钥明文存储、密码哈希算法强度不足、密钥轮换机制缺失等安全隐患。SecureAuthKeyModule模块旨在通过实现安全的用户认证机制、基于密码派生的密钥管理方案和完整的密钥轮换功能，为文件访问控制系统提供坚实的安全基础。

本模块基于Windows CNG（Cryptography Next Generation）密码学库实现，采用AES-256-GCM加密算法保护文件加密密钥，使用SHA-256哈希算法进行密码验证，通过PBKDF2衍生用户主密钥，确保密钥管理的安全性和可靠性。

### 1.3 范围

#### 1.3.1 系统范围

本模块主要包含以下功能范围：

- 用户注册与认证（用户名、密码验证）
- 用户登录与登出（主密钥加载与安全清除）
- 密码修改（含密钥自动轮换）
- 文件加密密钥（FEK）的创建、获取、删除
- 文件加密密钥（FEK）的加密存储（EFEK）
- 主密钥（Master Key）派生与管理
- 密钥轮换机制（修改密码时自动轮换所有FEK）

#### 1.3.2 排除范围

以下功能不在本模块范围内：

- 文件内容的实际加密和解密操作
- 网络通信安全
- 证书管理和SSL/TLS认证
- 用户角色和权限管理
- 审计日志记录

### 1.4 术语定义

- **FEK**：File Encryption Key，文件加密密钥，用于直接加密文件内容的AES-256密钥
- **EFEK**：Encrypted File Encryption Key，加密后的文件加密密钥，使用用户主密钥加密存储
- **Master Key**：主密钥，从用户密码派生的32字节密钥，用于加密FEK
- **CNG**：Cryptography Next Generation，Windows下一代密码学API
- **AES-256-GCM**：高级加密标准256位，伽罗瓦/计数器模式，提供加密和认证
- **SHA-256**：安全哈希算法256位，用于密码哈希和数据完整性验证
- **PBKDF2**：Password-Based Key Derivation Function 2，基于密码的密钥派生函数
- **Salt**：盐值，随机生成的16字节数据，用于增强密码哈希的安全性
- **Nonce**：一次性随机数，用于AES-GCM加密模式
- **Tag**：认证标签，AES-GCM模式下用于验证数据完整性和真实性

## 二、总体描述

### 2.1 用户特征

本模块面向文件访问控制系统的开发人员和集成人员，主要使用者包括：

- **系统开发者**：通过API接口集成模块到文件访问控制系统中
- **终端用户**：通过上层应用间接使用认证和密钥管理功能

### 2.2 运行环境

- 操作系统：Windows 10/11（64位）
- 开发语言：C++（Visual Studio 2022）
- 密码学库：Windows CNG（bcrypt.lib）
- 数据库：SQLite 3.0+
- 编译环境：MSVC 14.3+

### 2.3 约束条件

- **安全约束**：主密钥仅在内存中临时存储，登出时必须安全清除；密码必须使用加盐哈希存储；FEK必须加密存储。
- **性能约束**：密码哈希迭代次数为10000次，密钥派生时间不超过1秒。
- **技术约束**：使用Windows原生CNG API实现密码学功能，不依赖第三方密码学库。
- **数据约束**：数据库中不得存储明文密码和明文FEK。

## 三、功能性需求

### 3.1 用户认证模块

#### 3.1.1 用户注册

**用例编号**：UC-AUTH-001

**用例名称**：用户注册

**参与者**：系统开发者

**前置条件**：模块已初始化，数据库连接正常

**主事件流**：系统调用注册接口，传入用户名和密码；模块验证用户名和密码非空；检查用户名是否已存在；生成16字节随机盐值；将盐值与密码拼接后进行SHA-256哈希；将盐值和哈希结果组合存储到数据库；返回注册成功状态。

**备选事件流**：A1——用户名已存在，返回注册失败；A2——用户名或密码为空，返回参数错误；A3——哈希计算失败，返回系统错误。

**后置条件**：用户记录创建成功，密码哈希安全存储。

#### 3.1.2 用户登录

**用例编号**：UC-AUTH-002

**用例名称**：用户登录

**参与者**：系统开发者

**前置条件**：模块已初始化，数据库连接正常

**主事件流**：系统调用登录接口，传入用户名和密码；模块验证用户名和密码非空；检查是否已有登录用户，如有则先登出；从数据库获取存储的盐值和哈希；使用相同盐值对输入密码进行哈希；比对哈希值；验证通过后，使用PBKDF2从密码派生32字节主密钥；保存当前用户名和主密钥到内存；设置登录状态为已登录；返回登录成功状态。

**备选事件流**：A1——用户名不存在，返回登录失败；A2——密码哈希不匹配，返回登录失败；A3——主密钥派生失败，返回系统错误。

**后置条件**：用户登录成功，主密钥加载到内存，可进行密钥管理操作。

#### 3.1.3 用户登出

**用例编号**：UC-AUTH-003

**用例名称**：用户登出

**参与者**：系统开发者

**前置条件**：用户已登录

**主事件流**：系统调用登出接口；模块检查登录状态；使用SecureZeroMemory安全清除内存中的主密钥；清空当前用户名；设置登录状态为未登录。

**后置条件**：用户登出成功，内存中的敏感数据已安全清除。

#### 3.1.4 修改密码

**用例编号**：UC-AUTH-004

**用例名称**：修改密码

**参与者**：系统开发者

**前置条件**：模块已初始化，数据库连接正常

**主事件流**：系统调用修改密码接口，传入用户名、旧密码和新密码；模块验证参数非空；使用旧密码执行登录验证；登录成功后，执行密钥轮换操作（使用旧主密钥解密所有EFEK，使用新主密钥重新加密）；生成新密码的加盐哈希；更新数据库中的密码哈希；使用新密码派生新主密钥；保存新主密钥到内存；返回修改成功状态。

**备选事件流**：A1——旧密码验证失败，返回密码错误；A2——参数为空，返回参数错误；A3——密钥轮换失败，返回系统错误。

**后置条件**：密码更新成功，所有文件密钥已使用新主密钥重新加密。

### 3.2 密钥管理模块

#### 3.2.1 创建文件加密密钥

**用例编号**：UC-KEY-001

**用例名称**：创建文件加密密钥

**参与者**：系统开发者

**前置条件**：用户已登录，主密钥已加载

**主事件流**：系统调用创建密钥接口，传入文件ID；模块验证用户已登录且文件ID非空；生成32字节随机FEK（使用CNG随机数生成器）；使用主密钥和AES-256-GCM加密FEK，生成EFEK（包含Nonce、Tag、Ciphertext）；将EFEK存储到数据库（关联用户名和文件ID）；返回生成的FEK供上层使用。

**备选事件流**：A1——用户未登录，返回权限错误；A2——文件ID为空，返回参数错误；A3——随机数生成失败，返回系统错误；A4——加密失败，返回系统错误。

**后置条件**：FEK创建成功，EFEK安全存储到数据库，FEK返回给调用方用于文件加密。

#### 3.2.2 获取文件加密密钥

**用例编号**：UC-KEY-002

**用例名称**：获取文件加密密钥

**参与者**：系统开发者

**前置条件**：用户已登录，主密钥已加载

**主事件流**：系统调用获取密钥接口，传入文件ID；模块验证用户已登录且文件ID非空；从数据库查询该文件对应的EFEK；使用主密钥和AES-256-GCM解密EFEK，还原FEK；返回FEK供上层使用。

**备选事件流**：A1——用户未登录，返回权限错误；A2——文件ID为空，返回参数错误；A3——数据库中未找到该文件密钥，返回密钥不存在；A4——解密失败，返回系统错误。

**后置条件**：FEK获取成功，返回给调用方用于文件解密。

#### 3.2.3 删除文件加密密钥

**用例编号**：UC-KEY-003

**用例名称**：删除文件加密密钥

**参与者**：系统开发者

**前置条件**：用户已登录

**主事件流**：系统调用删除密钥接口，传入文件ID；模块验证用户已登录且文件ID非空；从数据库删除该文件对应的EFEK记录；返回删除成功状态。

**备选事件流**：A1——用户未登录，返回权限错误；A2——文件ID为空，返回参数错误；A3——删除失败，返回系统错误。

**后置条件**：文件密钥删除成功，数据库记录已清理。

#### 3.2.4 密钥轮换

**用例编号**：UC-KEY-004

**用例名称**：密钥轮换

**参与者**：系统开发者（内部调用）

**前置条件**：用户已登录，需要修改密码

**主事件流**：系统调用密钥轮换接口，传入用户名和新密码；模块从数据库获取该用户所有文件的EFEK列表；使用当前主密钥解密每个EFEK得到FEK；使用新密码派生新主密钥；使用新主密钥加密每个FEK生成新EFEK；更新数据库中的EFEK；安全清除旧主密钥和临时FEK；返回轮换成功状态。

**备选事件流**：A1——参数为空，返回参数错误；A2——数据库查询失败，返回系统错误；A3——解密或加密失败，跳过该密钥继续处理；A4——新主密钥派生失败，返回系统错误。

**后置条件**：所有文件密钥已使用新主密钥重新加密，旧主密钥已安全清除。

### 3.3 密码学工具模块

#### 3.3.1 密码哈希

**用例编号**：UC-CRYPTO-001

**用例名称**：密码哈希

**参与者**：系统开发者（内部调用）

**前置条件**：模块已初始化

**主事件流**：模块生成16字节随机盐值；将密码转换为UTF-8编码；将盐值与密码拼接；执行SHA-256哈希计算；返回盐值和哈希结果的组合数据。

**备选事件流**：A1——密码为空，返回空结果；A2——哈希计算失败，返回空结果。

**后置条件**：密码哈希计算完成，返回包含盐值和哈希的组合数据。

#### 3.3.2 密码验证

**用例编号**：UC-CRYPTO-002

**用例名称**：密码验证

**参与者**：系统开发者（内部调用）

**前置条件**：模块已初始化

**主事件流**：模块从存储的哈希数据中提取盐值（前16字节）和哈希（后32字节）；使用相同盐值对输入密码进行哈希计算；比对计算结果与存储的哈希；返回验证结果。

**备选事件流**：A1——存储数据格式不正确，返回验证失败；A2——密码为空，返回验证失败；A3——哈希计算失败，返回验证失败。

**后置条件**：密码验证完成，返回验证结果。

#### 3.3.3 随机数生成

**用例编号**：UC-CRYPTO-003

**用例名称**：随机数生成

**参与者**：系统开发者（内部调用）

**前置条件**：模块已初始化

**主事件流**：模块调用CNG随机数生成器生成指定长度的随机字节；若CNG调用失败，则使用C++标准库随机数生成器作为备选；返回生成的随机字节数组。

**备选事件流**：A1——CNG随机数生成失败，使用备选方案；A2——长度为0，返回空数组。

**后置条件**：随机数生成完成，返回指定长度的随机字节数组。

#### 3.3.4 FEK加密

**用例编号**：UC-CRYPTO-004

**用例名称**：FEK加密

**参与者**：系统开发者（内部调用）

**前置条件**：模块已初始化，主密钥和FEK有效

**主事件流**：模块验证主密钥长度为32字节、FEK长度为32字节；生成12字节随机Nonce；使用CNG AES-GCM模式，以主密钥加密FEK；生成16字节认证Tag；组合Nonce、Tag、Ciphertext生成EFEK；返回EFEK。

**备选事件流**：A1——主密钥或FEK长度不正确，返回失败；A2——CNG加密失败，返回失败。

**后置条件**：FEK加密完成，返回包含Nonce、Tag、Ciphertext的EFEK。

#### 3.3.5 FEK解密

**用例编号**：UC-CRYPTO-005

**用例名称**：FEK解密

**参与者**：系统开发者（内部调用）

**前置条件**：模块已初始化，主密钥和EFEK有效

**主事件流**：模块验证主密钥长度为32字节、EFEK长度至少28字节；从EFEK中提取Nonce（前12字节）、Tag（中间16字节）、Ciphertext（剩余部分）；使用CNG AES-GCM模式，以主密钥解密Ciphertext；验证认证Tag；返回解密后的FEK。

**备选事件流**：A1——主密钥或EFEK长度不正确，返回失败；A2——CNG解密失败，返回失败；A3——Tag验证失败，返回失败。

**后置条件**：FEK解密完成，返回原始FEK。

#### 3.3.6 主密钥派生

**用例编号**：UC-CRYPTO-006

**用例名称**：主密钥派生

**参与者**：系统开发者（内部调用）

**前置条件**：模块已初始化

**主事件流**：模块验证盐值长度至少16字节；将密码转换为UTF-8编码；将密码与盐值拼接；执行10000次SHA-256哈希迭代；返回32字节主密钥。

**备选事件流**：A1——盐值长度不足，返回空结果；A2——密码为空，返回空结果；A3——哈希计算失败，返回空结果。

**后置条件**：主密钥派生完成，返回32字节主密钥。

## 四、非功能性需求

### 4.1 安全性需求

- NF-SEC-001：主密钥仅在内存中临时存储，登出时必须使用SecureZeroMemory安全清除（高优先级）。
- NF-SEC-002：密码必须使用SHA-256加盐哈希存储，每个用户使用独立随机盐值（高优先级）。
- NF-SEC-003：FEK必须使用AES-256-GCM加密后存储，不得明文存储（高优先级）。
- NF-SEC-004：修改密码时必须自动轮换所有文件密钥，防止旧密码泄露导致密钥泄露（高优先级）。
- NF-SEC-005：使用CNG随机数生成器生成密钥和盐值，确保随机性（高优先级）。
- NF-SEC-006：AES-GCM加密必须验证认证Tag，防止数据篡改（高优先级）。

### 4.2 性能需求

- NF-PERF-001：密码哈希计算时间不超过1秒（中优先级）。
- NF-PERF-002：FEK加密/解密操作时间不超过100毫秒（中优先级）。
- NF-PERF-003：密钥轮换操作（100个文件密钥）时间不超过5秒（中优先级）。
- NF-PERF-004：数据库操作响应时间不超过200毫秒（中优先级）。

### 4.3 可靠性需求

- NF-RELI-001：CNG操作失败时应有备选方案（如随机数生成）（高优先级）。
- NF-RELI-002：密钥轮换过程中单个密钥失败不应影响其他密钥（高优先级）。
- NF-RELI-003：数据库操作应使用参数化查询，防止SQL注入（高优先级）。

## 五、数据需求

### 5.1 数据实体

#### 用户实体 (users)

用户实体包含以下属性：

- username：TEXT类型，主键，存储用户名。
- password_hash：TEXT类型，非空，存储密码哈希值（包含盐值和哈希结果的十六进制字符串）。

#### 文件密钥实体 (file_keys)

文件密钥实体包含以下属性：

- username：TEXT类型，主键之一，关联用户名。
- file_id：TEXT类型，主键之一，关联文件ID。
- encrypted_key：BLOB类型，非空，存储加密后的文件密钥（EFEK）。

### 5.2 数据关系

用户与文件密钥为一对多关系，一个用户可以拥有多个文件密钥。

### 5.3 数据完整性约束

- **主键约束**：users表username为主键；file_keys表(username, file_id)为联合主键。
- **非空约束**：password_hash和encrypted_key字段非空。

## 六、接口需求

### 6.1 类接口

#### AuthKeyManager类

AuthKeyManager类提供用户认证和密钥管理的统一接口：

- Initialize(const std::wstring& dbPath)：初始化模块，指定数据库文件路径，返回bool值表示成功与否。
- RegisterUser(const std::wstring& username, const std::wstring& password)：用户注册，返回bool值表示成功与否。
- Login(const std::wstring& username, const std::wstring& password)：用户登录，返回bool值表示成功与否。
- Logout()：用户登出，无返回值。
- ChangePassword(const std::wstring& username, const std::wstring& oldPassword, const std::wstring& newPassword)：修改密码，返回bool值表示成功与否。
- GetCurrentUsername()：获取当前登录用户名，返回std::wstring。
- CreateFileKey(const std::wstring& fileId, std::vector<BYTE>& outFEK)：创建文件密钥，返回bool值，outFEK参数输出生成的FEK。
- GetFileKey(const std::wstring& fileId, std::vector<BYTE>& outFEK)：获取文件密钥，返回bool值，outFEK参数输出解密后的FEK。
- DeleteFileKey(const std::wstring& fileId)：删除文件密钥，返回bool值。

#### CryptoUtils类

CryptoUtils类提供静态密码学工具方法：

- HashPasswordWithSalt(const std::wstring& password)：计算密码哈希，返回包含盐值和哈希的字节向量。
- VerifyPassword(const std::wstring& password, const std::vector<BYTE>& storedHashData)：验证密码，返回bool值。
- GenerateRandomBytes(size_t length)：生成随机字节，返回字节向量。
- EncryptFEK(const std::vector<BYTE>& masterKey, const std::vector<BYTE>& plaintextFEK, std::vector<BYTE>& outEFEK)：加密FEK，返回bool值。
- DecryptFEK(const std::vector<BYTE>& masterKey, const std::vector<BYTE>& EFEK, std::vector<BYTE>& outFEK)：解密FEK，返回bool值。
- DeriveMasterKey(const std::wstring& password, const std::vector<BYTE>& salt, int iterations)：派生主密钥，返回字节向量。

#### DatabaseManager类

DatabaseManager类提供数据库操作接口（内部使用）：

- Initialize(const std::wstring& dbPath)：初始化数据库连接，返回bool值。
- CreateUser(const std::wstring& username, const std::wstring& passwordHash)：创建用户记录，返回bool值。
- FindUser(const std::wstring& username, std::string& outPasswordHash)：查询用户，返回bool值，输出密码哈希。
- UpdatePassword(const std::wstring& username, const std::wstring& newPasswordHash)：更新密码，返回bool值。
- SaveFileKey(const std::wstring& username, const std::wstring& fileId, const std::vector<BYTE>& encryptedKey)：保存文件密钥，返回bool值。
- LoadFileKey(const std::wstring& username, const std::wstring& fileId, std::vector<BYTE>& outEncryptedKey)：加载文件密钥，返回bool值。
- DeleteFileKey(const std::wstring& username, const std::wstring& fileId)：删除文件密钥，返回bool值。
- GetAllFileKeys(const std::wstring& username, std::vector<std::pair<std::wstring, std::vector<BYTE>>>& outKeys)：获取用户所有文件密钥，返回bool值。
- UpdateFileKey(const std::wstring& username, const std::wstring& fileId, const std::vector<BYTE>& newEncryptedKey)：更新文件密钥，返回bool值。

### 6.2 外部接口

- **数据库接口**：SQLite数据库连接，使用sqlite3库实现数据持久化存储。
- **密码学接口**：Windows CNG API（bcrypt.lib），提供随机数生成、哈希计算、AES加密功能。

## 七、验收标准

### 7.1 功能验收标准

- UC-AUTH-001：新用户可成功注册，重复用户名注册失败。
- UC-AUTH-002：正确密码登录成功，错误密码登录失败。
- UC-AUTH-003：登出后内存中主密钥被安全清除。
- UC-AUTH-004：修改密码后所有文件密钥使用新主密钥重新加密。
- UC-KEY-001：成功创建FEK并加密存储为EFEK。
- UC-KEY-002：成功从数据库读取EFEK并解密为FEK。
- UC-KEY-003：成功删除指定文件的密钥记录。
- UC-KEY-004：密钥轮换后所有EFEK可使用新密码正确解密。
- UC-CRYPTO-001：密码哈希包含盐值和哈希结果。
- UC-CRYPTO-002：相同密码和盐值产生相同哈希，不同密码产生不同哈希。
- UC-CRYPTO-003：生成的随机字节具有足够随机性。
- UC-CRYPTO-004：加密后的FEK无法直接读取，解密后恢复原始FEK。
- UC-CRYPTO-005：解密验证Tag失败时返回错误。
- UC-CRYPTO-006：派生的主密钥长度为32字节。

### 7.2 非功能验收标准

- NF-SEC-001：登出后内存中的主密钥数组内容全为0。
- NF-SEC-002：数据库中密码字段为十六进制字符串，非明文。
- NF-SEC-003：数据库中file_keys表存储的是BLOB数据，无法直接读取FEK。
- NF-PERF-001：密码哈希计算时间小于1秒。
- NF-PERF-002：FEK加密/解密时间小于100毫秒。
- NF-RELI-003：使用SQL参数化查询，防止SQL注入攻击。
