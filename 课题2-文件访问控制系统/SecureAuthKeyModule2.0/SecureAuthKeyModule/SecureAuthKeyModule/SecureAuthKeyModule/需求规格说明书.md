## 一、引言

### 1.1 编写目的

本文档旨在详细描述SecureAuthKeyModule安全认证与密钥管理模块的功能需求、非功能需求、数据需求和接口需求，为系统设计、开发和测试提供明确的依据，确保模块满足信息系统安全课程设计的要求。

### 1.2 项目背景

在文件访问控制系统中，用户认证和密钥管理是保障系统安全的核心组件。传统的密钥管理方式存在密钥明文存储、密码哈希算法强度不足、密钥轮换机制缺失等安全隐患。SecureAuthKeyModule模块旨在通过实现安全的用户认证机制、基于密码派生的密钥管理方案和完整的密钥轮换功能，为文件访问控制系统提供坚实的安全基础。

本模块基于Windows CNG密码学库实现，采用AES-256-GCM加密算法保护文件加密密钥，使用SHA-256哈希算法进行密码验证，通过PBKDF2衍生用户主密钥，确保密钥管理的安全性和可靠性。

### 1.3 范围

#### 1.3.1 系统范围

本模块主要包含以下功能范围：

- 用户注册与认证（用户名、密码验证）
- 用户登录与登出（主密钥加载与安全清除）
- 密码修改（含密钥自动轮换）
- 文件加密密钥（FEK）的创建、获取、删除
- 文件加密密钥（FEK）的加密存储（EFEK）
- 主密钥（Master Key）派生与管理
- 密钥轮换机制（修改密码时自动轮换所有FEK）
- 文件加解密（AES-256-GCM文件级加密）
- 访问控制（基于角色的权限管理）
- 审计日志（操作记录与查询）
- USB端口管控（白名单机制与设备监控）

#### 1.3.2 排除范围

以下功能不在本模块范围内：

- 网络通信安全
- 证书管理和SSL/TLS认证
- 文件版本控制
- 分布式部署

### 1.4 术语定义

- **FEK**：File Encryption Key，文件加密密钥，用于直接加密文件内容的AES-256密钥
- **EFEK**：Encrypted File Encryption Key，加密后的文件加密密钥，使用用户主密钥加密存储
- **Master Key**：主密钥，从用户密码派生的32字节密钥，用于加密FEK
- **CNG**：Cryptography Next Generation，Windows下一代密码学API
- **AES-256-GCM**：高级加密标准256位，伽罗瓦/计数器模式，提供加密和认证
- **SHA-256**：安全哈希算法256位，用于密码哈希和数据完整性验证
- **PBKDF2**：Password-Based Key Derivation Function 2，基于密码的密钥派生函数
- **Salt**：盐值，随机生成的16字节数据，用于增强密码哈希的安全性
- **Nonce**：一次性随机数，用于AES-GCM加密模式
- **Tag**：认证标签，AES-GCM模式下用于验证数据完整性和真实性
- **RBAC**：Role-Based Access Control，基于角色的访问控制

## 二、总体描述

### 2.1 用户特征

本模块面向文件访问控制系统的开发人员和集成人员，主要使用者包括：

- **系统开发者**：通过API接口集成模块到文件访问控制系统中
- **终端用户**：通过上层应用间接使用认证和密钥管理功能

### 2.2 运行环境

- 操作系统：Windows 10/11（64位）
- 开发语言：C++（Visual Studio 2022）
- 密码学库：Windows CNG（bcrypt.lib）
- 数据库：SQLite 3.0+
- 编译环境：MSVC 14.3+

### 2.3 约束条件

- **安全约束**：主密钥仅在内存中临时存储，登出时必须安全清除；密码必须使用加盐哈希存储；FEK必须加密存储。
- **性能约束**：密码哈希迭代次数为10000次，密钥派生时间不超过1秒。
- **技术约束**：使用Windows原生CNG API实现密码学功能，不依赖第三方密码学库。
- **数据约束**：数据库中不得存储明文密码和明文FEK。

## 三、功能性需求

### 3.1 用户认证模块

#### 3.1.1 用户注册

**用例编号**：UC-AUTH-001

**用例名称**：用户注册

**参与者**：系统开发者

**前置条件**：模块已初始化，数据库连接正常

**主事件流**：系统调用注册接口，传入用户名和密码；模块验证用户名和密码非空；检查用户名是否已存在；生成16字节随机盐值；将盐值与密码拼接后进行SHA-256哈希；将盐值和哈希结果组合存储到数据库；返回注册成功状态。

**备选事件流**：A1——用户名已存在，返回注册失败；A2——用户名或密码为空，返回参数错误；A3——哈希计算失败，返回系统错误。

**后置条件**：用户记录创建成功，密码哈希安全存储。

#### 3.1.2 用户登录

**用例编号**：UC-AUTH-002

**用例名称**：用户登录

**参与者**：系统开发者

**前置条件**：模块已初始化，数据库连接正常

**主事件流**：系统调用登录接口，传入用户名和密码；模块验证用户名和密码非空；检查是否已有登录用户，如有则先登出；从数据库获取存储的盐值和哈希；使用相同盐值对输入密码进行哈希；比对哈希值；验证通过后，使用PBKDF2从密码派生32字节主密钥；保存当前用户名和主密钥到内存；设置登录状态为已登录；返回登录成功状态。

**备选事件流**：A1——用户名不存在，返回登录失败；A2——密码哈希不匹配，返回登录失败；A3——主密钥派生失败，返回系统错误。

**后置条件**：用户登录成功，主密钥加载到内存，可进行密钥管理操作。

#### 3.1.3 用户登出

**用例编号**：UC-AUTH-003

**用例名称**：用户登出

**参与者**：系统开发者

**前置条件**：用户已登录

**主事件流**：系统调用登出接口；模块检查登录状态；使用SecureZeroMemory安全清除内存中的主密钥；清空当前用户名；设置登录状态为未登录。

**后置条件**：用户登出成功，内存中的敏感数据已安全清除。

#### 3.1.4 修改密码

**用例编号**：UC-AUTH-004

**用例名称**：修改密码

**参与者**：系统开发者

**前置条件**：模块已初始化，数据库连接正常

**主事件流**：系统调用修改密码接口，传入用户名、旧密码和新密码；模块验证参数非空；使用旧密码执行登录验证；登录成功后，执行密钥轮换操作（使用旧主密钥解密所有EFEK，使用新主密钥重新加密）；生成新密码的加盐哈希；更新数据库中的密码哈希；使用新密码派生新主密钥；保存新主密钥到内存；返回修改成功状态。

**备选事件流**：A1——旧密码验证失败，返回密码错误；A2——参数为空，返回参数错误；A3——密钥轮换失败，返回系统错误。

**后置条件**：密码更新成功，所有文件密钥已使用新主密钥重新加密。

### 3.2 密钥管理模块

#### 3.2.1 创建文件加密密钥

**用例编号**：UC-KEY-001

**用例名称**：创建文件加密密钥

**参与者**：系统开发者

**前置条件**：用户已登录，主密钥已加载

**主事件流**：系统调用创建密钥接口，传入文件ID；模块验证用户已登录且文件ID非空；生成32字节随机FEK（使用CNG随机数生成器）；使用主密钥和AES-256-GCM加密FEK，生成EFEK（包含Nonce、Tag、Ciphertext）；将EFEK存储到数据库（关联用户名和文件ID）；返回生成的FEK供上层使用。

**备选事件流**：A1——用户未登录，返回权限错误；A2——文件ID为空，返回参数错误；A3——随机数生成失败，返回系统错误；A4——加密失败，返回系统错误。

**后置条件**：FEK创建成功，EFEK安全存储到数据库，FEK返回给调用方用于文件加密。

#### 3.2.2 获取文件加密密钥

**用例编号**：UC-KEY-002

**用例名称**：获取文件加密密钥

**参与者**：系统开发者

**前置条件**：用户已登录，主密钥已加载

**主事件流**：系统调用获取密钥接口，传入文件ID；模块验证用户已登录且文件ID非空；从数据库查询该文件对应的EFEK；使用主密钥和AES-256-GCM解密EFEK，还原FEK；返回FEK供上层使用。

**备选事件流**：A1——用户未登录，返回权限错误；A2——文件ID为空，返回参数错误；A3——数据库中未找到该文件密钥，返回密钥不存在；A4——解密失败，返回系统错误。

**后置条件**：FEK获取成功，返回给调用方用于文件解密。

#### 3.2.3 删除文件加密密钥

**用例编号**：UC-KEY-003

**用例名称**：删除文件加密密钥

**参与者**：系统开发者

**前置条件**：用户已登录

**主事件流**：系统调用删除密钥接口，传入文件ID；模块验证用户已登录且文件ID非空；从数据库删除该文件对应的EFEK记录；返回删除成功状态。

**备选事件流**：A1——用户未登录，返回权限错误；A2——文件ID为空，返回参数错误；A3——删除失败，返回系统错误。

**后置条件**：文件密钥删除成功，数据库记录已清理。

#### 3.2.4 密钥轮换

**用例编号**：UC-KEY-004

**用例名称**：密钥轮换

**参与者**：系统开发者（内部调用）

**前置条件**：用户已登录，需要修改密码

**主事件流**：系统调用密钥轮换接口，传入用户名和新密码；模块从数据库获取该用户所有文件的EFEK列表；使用当前主密钥解密每个EFEK得到FEK；使用新密码派生新主密钥；使用新主密钥加密每个FEK生成新EFEK；更新数据库中的EFEK；安全清除旧主密钥和临时FEK；返回轮换成功状态。

**备选事件流**：A1——参数为空，返回参数错误；A2——数据库查询失败，返回系统错误；A3——解密或加密失败，跳过该密钥继续处理；A4——新主密钥派生失败，返回系统错误。

**后置条件**：所有文件密钥已使用新主密钥重新加密，旧主密钥已安全清除。

## 四、非功能性需求

### 4.1 安全性需求

- NF-SEC-001：主密钥仅在内存中临时存储，登出时必须使用SecureZeroMemory安全清除（高优先级）。
- NF-SEC-002：密码必须使用SHA-256加盐哈希存储，每个用户使用独立随机盐值（高优先级）。
- NF-SEC-003：FEK必须使用AES-256-GCM加密后存储，不得明文存储（高优先级）。
- NF-SEC-004：修改密码时必须自动轮换所有文件密钥，防止旧密码泄露导致密钥泄露（高优先级）。
- NF-SEC-005：使用CNG随机数生成器生成密钥和盐值，确保随机性（高优先级）。
- NF-SEC-006：AES-GCM加密必须验证认证Tag，防止数据篡改（高优先级）。
- NF-SEC-007：访问控制必须基于角色和直接授权双重验证，防止越权访问（高优先级）。
- NF-SEC-008：所有操作必须记录审计日志，确保可追溯性（高优先级）。
- NF-SEC-009：USB设备必须通过白名单验证，防止未授权设备接入（高优先级）。

### 4.2 性能需求

- NF-PERF-001：密码哈希计算时间不超过1秒（中优先级）。
- NF-PERF-002：FEK加密/解密操作时间不超过100毫秒（中优先级）。
- NF-PERF-003：密钥轮换操作（100个文件密钥）时间不超过5秒（中优先级）。
- NF-PERF-004：数据库操作响应时间不超过200毫秒（中优先级）。
- NF-PERF-005：文件加解密速度不低于1MB/s（中优先级）。

### 4.3 可靠性需求

- NF-RELI-001：CNG操作失败时应有备选方案（如随机数生成）（高优先级）。
- NF-RELI-002：密钥轮换过程中单个密钥失败不应影响其他密钥（高优先级）。
- NF-RELI-003：数据库操作应使用参数化查询，防止SQL注入（高优先级）。
- NF-RELI-004：审计日志记录成功率不低于99.9%（高优先级）。

## 五、数据需求

### 5.1 数据实体

#### 用户实体 (users)

用户实体包含以下属性：

- username：TEXT类型，主键，存储用户名。
- password_hash：TEXT类型，非空，存储密码哈希值。

#### 文件密钥实体 (file_keys)

文件密钥实体包含以下属性：

- username：TEXT类型，主键之一，关联用户名。
- file_id：TEXT类型，主键之一，关联文件ID。
- encrypted_key：BLOB类型，非空，存储加密后的文件密钥。

#### 角色实体 (roles)

角色实体包含以下属性：

- role_name：TEXT类型，主键，存储角色名称。
- description：TEXT类型，存储角色描述。

#### 用户角色实体 (user_roles)

用户角色实体包含以下属性：

- username：TEXT类型，主键之一，关联用户名。
- role_name：TEXT类型，主键之一，关联角色名称。

#### 文件权限实体 (file_permissions)

文件权限实体包含以下属性：

- username：TEXT类型，主键之一，关联用户名。
- file_id：TEXT类型，主键之一，关联文件ID。
- permissions：INTEGER类型，非空，存储权限标志位。

#### 文件所有者实体 (file_owners)

文件所有者实体包含以下属性：

- file_id：TEXT类型，主键，关联文件ID。
- owner：TEXT类型，非空，存储所有者用户名。

#### 审计日志实体 (audit_logs)

审计日志实体包含以下属性：

- id：INTEGER类型，主键，自增。
- username：TEXT类型，非空，关联用户名。
- action：TEXT类型，非空，存储操作类型。
- file_id：TEXT类型，可空，关联文件ID。
- file_path：TEXT类型，可空，存储文件路径。
- timestamp：TIMESTAMP类型，非空，存储操作时间。
- success：INTEGER类型，非空，存储操作结果。

#### USB白名单实体 (usb_whitelist)

USB白名单实体包含以下属性：

- vid：TEXT类型，主键之一，存储厂商ID。
- pid：TEXT类型，主键之一，存储产品ID。
- serial_number：TEXT类型，主键之一，存储设备序列号。
- device_name：TEXT类型，存储设备名称。
- authorized：INTEGER类型，非空，存储授权状态。

#### USB日志实体 (usb_logs)

USB日志实体包含以下属性：

- id：INTEGER类型，主键，自增。
- vid：TEXT类型，存储厂商ID。
- pid：TEXT类型，存储产品ID。
- serial_number：TEXT类型，存储设备序列号。
- device_name：TEXT类型，存储设备名称。
- action：TEXT类型，非空，存储操作类型。
- success：INTEGER类型，非空，存储操作结果。
- timestamp：TIMESTAMP类型，非空，存储操作时间。

### 5.2 数据关系

用户与文件密钥为一对多关系；用户与用户角色为一对多关系；角色与用户角色为一对多关系；用户与文件权限为一对多关系；文件与文件权限为一对多关系；文件与文件所有者为一对一关系；用户与审计日志为一对多关系。

### 5.3 数据完整性约束

- **主键约束**：各实体主键唯一且非空。
- **外键约束**：关联表外键引用主表主键。
- **唯一约束**：用户名唯一；角色名称唯一。
- **非空约束**：必要字段非空。
- **默认值约束**：权限默认值为0，授权状态默认值为0。
