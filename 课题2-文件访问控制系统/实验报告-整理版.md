# 文件访问控制系统实验报告

---

## 一、概述

### 项目背景

在信息化时代，文件安全管理是企业和组织面临的重要挑战。传统的文件管理方式存在权限管理混乱、访问控制缺失、操作无记录等安全隐患。文件访问控制系统旨在通过统一的身份认证、细粒度的权限管理和完整的审计日志，实现对文件资源的安全管控。

本项目设计并实现了一个基于角色的文件访问控制系统(RBAC)，支持用户注册登录、文件上传下载、权限管理、用户管理和审计日志等核心功能，为企业提供安全可靠的文件管理解决方案。

### 项目意义

1. 实现基于角色的访问控制(RBAC)机制
2. 提供完整的用户认证和授权功能
3. 实现细粒度的文件权限管理
4. 记录完整的操作审计日志
5. 增强文件访问的安全性和可追溯性
6. 培养信息系统安全设计和开发能力

### 功能模块

系统分为以下六个核心模块：

1. **用户认证模块**：负责用户注册、登录、密码管理和会话验证
2. **角色权限模块**：定义角色权限映射，实现访问控制
3. **文件管理模块**：实现文件上传、下载、删除和权限设置
4. **用户管理模块**：管理员对用户进行管理和维护
5. **审计日志模块**：记录所有用户操作，支持安全审计
6. **数据库模块**：提供数据持久化存储

### 角色权限定义

| 角色 | 权限 | 说明 |
| :--- | :--- | :--- |
| admin | read, write, delete, manage_users, audit | 系统管理员，拥有全部权限 |
| manager | read, write, delete, audit | 管理者，可管理文件和查看日志 |
| user | read, write | 普通用户，可上传和下载文件 |
| guest | read | 访客，仅可查看公开文件 |

5. 判断用户是否拥有访问权限

```python
import hashlib
import uuid
import time

def login(username, password, ip_address):
    cursor.execute('SELECT id, password_hash, salt FROM users WHERE username = ?', (username,))
    user = cursor.fetchone()
    
    if hash_password(password, salt) != password_hash:
        return None, '密码错误'
    
    session_id = str(uuid.uuid4())
    expires_at = int(time.time()) + 3600
    
    cursor.execute('INSERT INTO sessions (id, user_id, expires_at) VALUES (?, ?, ?)', 
                   (session_id, user_id, expires_at))
    return {'session_id': session_id}, None

def hash_password(password, salt):
    return hashlib.sha256((password + salt).encode()).hexdigest()
```

#### 权限检查核心代码

```python
ROLES = {
    'admin': ['read', 'write', 'delete', 'manage_users', 'audit'],
    'manager': ['read', 'write', 'delete', 'audit'],
    'user': ['read', 'write'],
    'guest': ['read']
}

def check_permission(user_role, required_permission):
    if user_role not in ROLES:
        return False
    return required_permission in ROLES[user_role]

def has_file_access(user_id, file_id, required_permission):
    cursor.execute('SELECT owner_id, permission FROM files WHERE id = ?', (file_id,))
    file = cursor.fetchone()
    
    if user_id == file[0]:
        return True
    
    if file[1] == 'public' and required_permission == 'read':
        return True
    
    cursor.execute('SELECT permission FROM file_permissions WHERE file_id = ? AND user_id = ?', 
                   (file_id, user_id))
    permission = cursor.fetchone()
    
    return permission and required_permission in permission[0]
```

### 安全测试

#### 密码安全
- 密码使用SHA256加盐哈希存储
- 每个用户使用独立的随机盐值
- 无法从哈希值反推原始密码

#### 会话安全
- 会话ID使用UUID生成
- 会话有效期1小时
- 登录成功后生成新会话
- 修改密码后自动失效所有会话

#### 权限控制
- 基于角色的访问控制(RBAC)
- 细粒度的文件权限检查
- 防止越权访问

## 六、总结

### 项目成果

本项目成功实现了一个完整的文件访问控制系统，主要成果包括：

1. **用户认证系统**：实现了安全的用户注册、登录和密码管理功能，密码使用SHA256加盐哈希存储
2. **角色权限管理**：基于RBAC模型，定义了四种角色，实现细粒度的访问控制
3. **文件管理功能**：支持文件上传、下载、删除，限制文件大小和类型
4. **文件权限控制**：支持私有/公开权限设置和自定义用户授权
5. **用户管理**：管理员可编辑用户信息、启用/禁用用户、重置密码
6. **审计日志**：完整记录所有用户操作，支持安全审计
7. **Web界面**：提供友好的图形化操作界面

### 关键技术难点与解决方案

| 难点 | 解决方案 |
| :--- | :--- |
| 密码安全存储 | 使用SHA256哈希算法，每个用户使用独立随机盐值 |
| 会话管理 | 使用UUID生成会话ID，设置合理的过期时间 |
| 权限检查 | 实现基于角色和资源的双重权限检查 |
| 文件完整性 | 使用SHA256计算文件哈希值 |
| 防止越权 | 在每个操作前进行权限验证 |

### 安全意识提升

通过本项目的实践，深刻理解了信息系统安全的核心概念：
- 身份认证的重要性和实现方法

- 密码安全存储的最佳实践
